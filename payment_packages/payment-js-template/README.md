# Payment JS Template

AuraLogic `payment_js` package starter sample.

This sample is intentionally conservative:

- plain JavaScript entry with ES5-friendly syntax
- package manifest with compatibility metadata
- package-governed `config_schema`
- secure webhook declaration backed by a config secret
- polling callback + refund callback example
- local manifest validation, smoke tests, and ZIP packaging

## Files

- `manifest.json` - payment package manifest
- `src/index.js` - runtime callbacks for card generation, status polling, webhook handling, and refund
- `scripts/build.mjs` - copies the runtime entry into `dist/`
- `scripts/test.mjs` - smoke-tests the built package in a Node VM with mocked AuraLogic APIs
- `scripts/package.mjs` - creates `payment-js-template.zip`

## Commands

1. `make deps`
2. `npm run validate:manifest`
3. `npm run test`
4. `npm run package`

## Upload flow

1. Run `npm run package`
2. Upload `payment-js-template.zip` through `/admin/payment-methods`
3. Open the created payment method and review the package-governed config defaults
4. Use the declared callback URL `/api/payment-methods/{id}/webhooks/payment.notify`

## Demo webhook request

```bash
curl -X POST "http://localhost:8080/api/payment-methods/1/webhooks/payment.notify" \
  -H "Content-Type: application/json" \
  -H "X-Payment-Webhook-Token: change-me-demo-token" \
  -d '{"order_no":"ORD-1001","status":"paid","transaction_id":"TX-DEMO-1001"}'
```

What the sample demonstrates:

- `onGeneratePaymentCard(order, config)`
  - renders a simple checkout card
  - shows the package-declared callback path generated by `AuraLogic.system.getWebhookUrl(...)`
  - records lightweight payment metadata through `AuraLogic.order.updatePaymentData(...)`
- `onCheckPaymentStatus(order, config)`
  - checks whether a prior webhook already stored a paid record
  - optionally supports `simulate_paid=true` for local smoke testing
- `onWebhook(hook, config)`
  - reads raw request data from `AuraLogic.webhook`
  - validates the package-declared callback mode
  - returns `paid/order_no/transaction_id` for direct confirmation, or `queue_polling=true` when the package wants the host to poll later
- `onRefund(order, config)`
  - records a refund request in storage and returns a manual-processing response

## Notes

- The host currently expects `cache_ttl` for payment card caching.
- The declared webhook secret comes from `config.webhook_token` because payment package webhook auth reads secrets from the payment method config JSON.
- For production packages, replace the demo HTML and callback parsing with your provider-specific flow.
- For runtime API details, see `docs/PAYMENT_JS_API.md`.
